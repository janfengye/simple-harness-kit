# Project Constraints

**SINGLE SOURCE OF TRUTH** — 所有规则的唯一权威来源。

## 约束 ID 格式

- `C-{area}-{number}` — 单条约束
- `JC-{number}` — 联合约束组（组内必须同时成立）
- `VH-{number}` — 违规历史

## 约束区域（area）

- `UI` — 界面 / `A11Y` — 无障碍 / `PERF` — 性能
- `SEC` — 安全 / `TEST` — 测试 / `ARCH` — 架构
- `SEARCH` — 搜索功能 / `FILTER` — 筛选功能

---

## [JC-01: 搜索与筛选交互]

| ID | 约束 | WHY | 违反后果 |
|---|---|---|---|
| C-SEARCH-01 | 搜索框必须有 aria-label 和 role="search" | 无障碍标准要求 | a11y 测试失败 |
| C-SEARCH-02 | 搜索为实时过滤，无需提交按钮 | 用户体验要求 | 交互不符合 spec |
| C-SEARCH-03 | 无结果时必须显示空状态提示 | 用户需要反馈 | 用户困惑 |
| C-FILTER-01 | 搜索和标签筛选必须可组合使用 | 核心交互需求 | 功能不完整 |
| C-FILTER-02 | "All" 标签清除所有筛选条件 | 用户需要重置途径 | 无法恢复全部视图 |

## [JC-02: 响应式与无障碍]

| ID | 约束 | WHY | 违反后果 |
|---|---|---|---|
| C-UI-01 | 搜索框在移动端（<640px）必须全宽显示 | 响应式设计要求 | 移动端体验差 |
| C-A11Y-01 | 所有表单元素必须有关联的 label | WCAG 2.1 AA | axe 扫描失败 |
| C-A11Y-02 | 筛选按钮状态变化必须有 aria-pressed 属性 | 屏幕阅读器需要 | a11y 不达标 |

## [JC-03: 测试覆盖]

| ID | 约束 | WHY | 违反后果 |
|---|---|---|---|
| C-TEST-01 | 新功能必须有对应的 E2E 测试 | 质量保证 | 回归风险 |
| C-TEST-02 | E2E 测试必须覆盖搜索+筛选组合场景 | 交互组合是高风险区域 | 组合 bug 逃逸 |
| C-TEST-03 | 现有 E2E 测试不可因新功能而失败 | 回归保护 | 引入新问题 |
