# 场景 6：Session Log 记录质量

## 前置条件
- 项目已初始化 Harness

## 指令

执行任意功能开发（如场景 2 的 truncate 函数），完成后检查 session-log。

## 验证清单

- [ ] 有真实时间戳（非 `[--:--]` 占位符）
- [ ] 记录了阶段切换（如"阶段切换: PLAN → EXECUTE"）
- [ ] 记录了人的指示原话
- [ ] 记录了 AI 的关键决策和依据
- [ ] 记录了 Hook 事件（如有触发）
- [ ] 记录了 Gate 判定结果
- [ ] 偏差（如有）被记录（方法论要求 X，实际做了 Y）

## 回归风险

Session Log 是方法论自我改进的数据来源。如果记录质量差，后续无法分析优化。
