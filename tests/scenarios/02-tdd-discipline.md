# 场景 2：TDD 纪律

## 前置条件
- 项目已初始化 Harness

## 指令

```
帮我实现一个字符串工具函数：truncate(str, maxLen)
截断字符串到指定长度，超出部分用 ... 替换。
```

## 验证清单

- [ ] 先写测试再实现（session-log 时序可验证）
- [ ] 测试先失败后通过（RED→GREEN）
- [ ] 测试覆盖：正常截断、不需要截断、空字符串、maxLen=0
- [ ] 实现代码是通过测试的最少代码（YAGNI）
- [ ] 不添加超出需求的额外功能

## 回归风险

TDD 是 Layer 1 的基础。如果此场景失败，整个 QA 金字塔根基不稳。
